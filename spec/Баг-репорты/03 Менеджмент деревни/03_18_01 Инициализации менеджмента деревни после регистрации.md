# Баг-репорт - Инициализации менеджмента деревни после регистрации

**Статус:** Открыт  
**Приоритет:** Высокий  
**Дата начала:** 15.11.2025  
**Дата окончания:**    
**Environment:** Google Chrome (последняя версия)  

## Описание

При регистрации нового аккаунта и первом входе в игру через кнопку "В игру" возникает ошибка 242 "params not set fully". Деревня не отображается корректно (отсутствуют Шахта и Ратуша), интерфейс частично не функционирует. После принудительного обновления страницы и повторного входа в аккаунт появляется ошибка 9000, но деревня начинает работать стабильно.

## Предусловия

- Пользователь зарегистраовал новый аккаунт
- Выполнен первый вход в игру после регистрации

## Шаги воспроизведения

1. Завершить регистрацию нового аккаунта в системе
2. Нажать кнопку "В игру" для первого входа в деревню
3. Наблюдать за отображением игрового интерфейса и сообщениями об ошибках
4. Попытаться взаимодействовать с элементами управления

## Фактический результат

- Появляется ошибка 242 "params not set fully"
- Деревня не отображается корректно (отсутствуют Шахта и Ратуша)
- Кнопки "village" и "выход" не реагируют на нажатия
- После принудительного обновления страницы и повторного входа:
  - Появляется ошибка 9000
  - Деревня отображается корректно (Шахта и Ратуша присутствуют)
  - Игровой процесс функционирует стабильно

## Ожидаемый результат

- После регистрации нового аккаунта и нажатия кнопки "В игру" деревня должна загружаться корректно с первого раза
- Все базовые здания (Шахта, Ратуша) должны отображаться сразу
- Интерфейс должен быть полностью функциональным без необходимости перезагрузки страницы
- Отсутствие ошибок 242 и 9000 при первом входе
- Пользователь может сразу начать взаимодействовать с деревней и строить здания (в пределах доступных средств)
